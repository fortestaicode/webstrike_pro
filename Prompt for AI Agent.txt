Prompt for AI Agent:
Role: You are an expert Python developer specializing in web scraping, ethical hacking tools, and cross-platform automation scripts. Your task is to create a production-ready, multi-threaded website mirroring and reconnaissance tool.
Task: Create a Python script named web_recon.py that performs complete website mirroring (downloading all files while preserving directory structure), discovers hidden files/directories using a wordlist, and implements advanced anti-detection mechanisms.
Core Requirements:
Cross-Platform Compatibility: Must work flawlessly on both Windows and Linux (handle path separators, file naming restrictions, encoding issues).
Hierarchical Directory Preservation: Mirror the exact directory structure of the target website locally. Convert URLs to local paths correctly (e.g., https://site.com/blog/post.php → mirror/site.com/blog/post.php).
Smart File Discovery & Download:
Crawl all discovered HTML pages and extract links (href, src, url() in CSS).
Download all assets (CSS, JS, Images, Fonts, PDFs, etc.).
Intelligent PHP Handling:
By default, do NOT download .php files unless the server leaks source code.
Detection Logic: If a .php file returns content containing PHP markers (<?php, <?=, <?), or PHP syntax patterns (function, class, $variable, echo, include), then download it as it indicates source code exposure.
Always download PHP backup variants regardless: .php.bak, .php~, .php.old, .php.swp, .php.save, .php.txt, .php.zip.
Support extension exclusion via -e flag (comma-separated, e.g., -e php,css skips these extensions unless they are backup variants).
Hidden Files Discovery (Reconnaissance):
Read wordlist from file specified by -l flag.
For every directory discovered during crawling, brute-force hidden files using the wordlist (append wordlist entries to directory paths).
Example: If /blog/ is found, test /blog/.env, /blog/config.php, /blog/backup.zip, etc.
Use multi-threading for this operation.
Anti-Detection & Stealth Features (Mandatory):
Rotating User-Agents: Maintain a pool of 20+ real browser User-Agents (Chrome, Firefox, Safari on Windows/Mac/Linux). Rotate them frequently (every 3-5 requests).
Request Headers: Randomize Accept, Accept-Language, Accept-Encoding, Referer (set referer to the parent page or homepage).
Request Delays: Implement random delays between requests (jitter) within the range specified.
Session Management: Use requests.Session() with cookie persistence but clear cookies periodically to avoid tracking.
Timeout Handling: Configurable connection and read timeouts.
Multi-threading: Use ThreadPoolExecutor for concurrent downloads. Separate thread pools for crawling and hidden file discovery to optimize performance.
Command Line Interface:
bash
Copy
python web_recon.py <URL> [OPTIONS]

Options:
-e, --exclude       Comma-separated extensions to exclude (e.g., php,css,js). Default: none (except applied smart logic).
-l, --wordlist      Path to wordlist file for hidden files discovery. Default: built-in common list.
-t, --threads       Number of concurrent threads. Default: 10.
-d, --delay         Format: "delay,timeout" (e.g., 0.1,2). Delay between requests (seconds) and request timeout (seconds). Default: 0.1,2.
-m, --max-files     Maximum number of files to download (0 or -1 for unlimited). Default: unlimited.
-o, --output        Output directory name. Default: "mirror".
--max-file-size     Skip files larger than X MB (default: 100MB). Set 0 to disable.
--depth             Maximum crawl depth (0 for unlimited). Default: 0.
-v, --verbose       Verbose output.
Advanced Features (Make it Professional):
Resume Capability: Check if file exists locally with same size before re-downloading (unless --force used).
Retry Logic: Exponential backoff for failed requests (5xx errors, timeouts). Max 3 retries.
MIME-Type Validation: Verify content-type matches extension, handle misconfigured servers.
Rate Limit Detection: If HTTP 429 (Too Many Requests) received, pause for 60 seconds then resume.
Dynamic URL Handling: Handle relative URLs, absolute URLs, protocol-relative URLs (//), and query parameters (save files with hashed query strings to avoid collisions).
Smart Duplicate Prevention: Use sets to track visited URLs (normalized).
Link Rewriting: Rewrite HTML/CSS links to work locally (convert absolute URLs to relative paths).
